Reactive streams

  • SPI, not intended to be user-facing
  • Provides interop among implementations

trait Publisher[T] {
    def subscribe(subscriber: Subscriber[T]): Unit
}

trait Subscriber[T] {
    def onSubscribe(sub: Subscription): Unit
    def onNext(t: T): Unit
    def onError(t: Throwable): Unit
    def onComplete()
}

trait Subscription {
    def request(n: Long): Unit
    def cancel(): Unit
}
    Connect flow
  • Sub => Pub subscribe
  • Pub => Sub onSubscribe
    • Acknowledgement
    Data flow
  • Sub => Pub request (via Subscription)
    • Upstream demand signaling
  • Pub => Sub `onNext`
    • Downstream data flow
    Completion
  • Pub => Sub onComplete
  • Pub => Sub onError
  • Sub => (via Subscription) => Pub cancel
